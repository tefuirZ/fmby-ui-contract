# Manage · Pan115

115 原生能力在当前主线分成两条线：

- `pan115`：绑定账号后的原生网盘挂载
- `pan115-share`：基于分享链接 / 提取码的公开分享挂载

二者都属于 WebUI 管理域的一部分，但目录浏览和根路径语义不同，皮肤不能混用。

## `pan115` 端点

| Method | Path | 说明 |
|--------|------|------|
| POST | `/api/manage/pan115/qr-login` | 申请扫码登录 |
| GET | `/api/manage/pan115/qr-status` | 轮询扫码状态 |
| POST | `/api/manage/pan115/activate` | 激活 mount 或创建态预览 |
| POST | `/api/manage/pan115/previews` | 创建临时预览会话 |
| POST | `/api/manage/pan115/previews/{previewId}/browse` | 创建态目录浏览 |
| GET | `/api/manage/pan115/accounts/{mountId}` | 查看账号状态 |
| POST | `/api/manage/pan115/accounts/{mountId}/refresh` | 刷新 Open token |
| POST | `/api/manage/pan115/accounts/{mountId}/health` | 主动健康检查 |
| DELETE | `/api/manage/pan115/accounts/{mountId}` | 解绑账号 |
| POST | `/api/manage/pan115/accounts/{mountId}/browse` | 编辑态目录浏览 |

## `pan115` 核心语义

### 1. 根路径保存的是目录 `cid`

`pan115` 挂载的 `root_path` 不是普通路径文本，而是 115 目录 `cid` 字符串，例如：

```jsonc
{
  "provider_type": "Pan115",
  "root_path": "123456789",
  "config_json": {
    "client_type": "ios",
    "open_app_ids": [100195863, 100195864],
    "request_policy": {
      "request_timeout_ms": 30000,
      "min_request_interval_ms": 500
    }
  }
}
```

### 2. 目录浏览必须走专用 `pan115` browse API

不要把普通 `browse-directories` 当成 `pan115` 的唯一真相。`pan115` 依赖已激活账号 runtime，WebUI 与第三方主题都应这样接：

- 创建挂载时：`/api/manage/pan115/previews/{previewId}/browse`
- 编辑挂载或媒体库绑定时：`/api/manage/pan115/accounts/{mountId}/browse`

返回：

```jsonc
{
  "mount_id": "mount_xxx",
  "current_path": "/电影/4K",
  "parent_path": "/电影",
  "current_cid": "99887766",
  "entries": [
    {
      "name": "动作片",
      "path": "/电影/4K/动作片",
      "cid": "99887767",
      "is_dir": true,
      "size": null,
      "modified_at": "2026-05-11T18:10:00Z"
    }
  ]
}
```

### 3. 媒体库绑定子路径仍然是相对路径

虽然挂载根是 `cid`，但媒体库绑定 `sub_path` 仍然保存为相对 mount 根的逻辑路径，例如：

```jsonc
{
  "mount_id": "mount_pan115_movies",
  "sub_path": "电影/4K",
  "scan_priority": 10
}
```

也就是说：

- `root_path`：挂载级别的真实 115 根目录 `cid`
- `sub_path`：媒体库绑定级别的逻辑子目录路径

## `pan115-share` 契约

`pan115-share` 不走账号密文表，直接依赖分享链接工作。

### 创建 payload

```jsonc
{
  "provider_type": "Pan115Share",
  "root_path": "/电影合集/4K",
  "config_json": {
    "share_code": "abc123",
    "receive_code": "qwer",
    "client_type": "chrome",
    "request_policy": {
      "request_timeout_ms": 30000,
      "min_request_interval_ms": 500
    }
  }
}
```

### 关键语义

1. `share_code` 支持完整分享链接或纯 code，前端可自动提取。
2. `root_path` 保存的是“分享内子目录根”，不是 share cid。
3. `receive_code` 对公开分享可留空，对加密分享必须支持填写。
4. `client_type` 决定下载链路通道，默认 `chrome`。

### 目录浏览

`pan115-share` 走通用 `/api/manage/mounts/browse-directories`，但媒体库绑定时必须传入既有 mount 的 `root_path`，表示“在分享根内继续选择子目录”。

## `request_policy`

115 相关 provider 与其它远端源保持统一：前端只写 `config_json.request_policy`。

当前主线真实消费边界：

- `pan115`：runtime 注入时消费 `request_timeout_ms`、`min_request_interval_ms`
- `pan115-share`：分享目录浏览与下载链路消费 `request_timeout_ms`、`min_request_interval_ms`

`max_retries` / `cooldown` 等其它治理字段目前仍是前向兼容字段，主题包可以展示，但不能声称当前版本后端已全部消费。

## 115 图床入口

115 图床子系统独立见 [pan115-imghost.md](./pan115-imghost.md)。

当前 WebUI 入口状态：

- 前端 feature flag 常量：`PAN115_IMGHOST_ENABLED = true`
- 主题或第三方皮肤仍应以站点 / 产品配置作为最终展示门禁，不要只依赖硬编码常量

## 皮肤实现建议

1. `pan115` 创建流优先引导扫码或导入 Cookie，再浏览目录拿 `cid`，不要让管理员手填。
2. `pan115-share` 创建流先填分享链接 / 提取码，再浏览分享目录拿 `root_path`。
3. 媒体库绑定页必须提供目录浏览器，不能要求管理员手输远端子路径。
4. 详情页应显式区分：
   - `pan115`：账号状态、健康状态、刷新 token、解绑
   - `pan115-share`：分享配置、下载通道、分享子目录根、远端治理
