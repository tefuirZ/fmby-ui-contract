# Manage · Pan115

115 在当前主线分成三条互相隔离的能力线：

- `pan115`：我的网盘挂载。Cookie 负责目录索引和运行时注入；Open 负责账号信息、下载和播放临时链接。
- `pan115-share`：分享数据源。分享目录导入不依赖 Cookie；播放、技术探测、sidecar 读取和下载临时链接才使用 mount 级分享 Cookie。
- `pan115-imghost`：图床镜像子系统。凭据独立于挂载凭据，见 [pan115-imghost.md](./pan115-imghost.md)。

皮肤实现必须把这三条线分开，不允许做成“115 全局账号池”或“全局分享 Cookie”。

## `pan115` 端点

| Method | Path | 说明 |
|--------|------|------|
| POST | `/api/manage/pan115/qr-login` | 申请扫码登录 |
| GET | `/api/manage/pan115/qr-status?session_id=` | 轮询扫码状态 |
| POST | `/api/manage/pan115/previews` | 将扫码会话转成创建态短 TTL 预览 |
| POST | `/api/manage/pan115/previews/{previewId}/browse` | 创建态目录浏览 |
| POST | `/api/manage/pan115/activate` | 绑定到已创建 mount；支持 `preview_id` |
| GET | `/api/manage/pan115/accounts/{mountId}` | 查看账号状态 |
| POST | `/api/manage/pan115/accounts/{mountId}/refresh` | 刷新 Open token |
| POST | `/api/manage/pan115/accounts/{mountId}/health` | 主动健康检查 |
| DELETE | `/api/manage/pan115/accounts/{mountId}` | 解绑账号 |
| POST | `/api/manage/pan115/accounts/{mountId}/browse` | 编辑态目录浏览 |
| GET | `/api/manage/pan115/sync/mounts/{mountId}` | 查看 raw sync 概览 |
| POST | `/api/manage/pan115/sync/mounts/{mountId}/enqueue` | 手动入队 full / life / diff 等同步任务 |

## `pan115` 核心语义

### 根路径保存目录 `cid`

`pan115` 挂载的 `root_path` 不是普通路径文本，而是 115 目录 `cid` 字符串，例如：

```jsonc
{
  "provider_type": "Pan115",
  "root_path": "123456789",
  "config_json": {
    "client_type": "ios",
    "open_app_ids": [100195863, 100195864],
    "request_policy": {
      "request_timeout_ms": 30000,
      "min_request_interval_ms": 500
    }
  }
}
```

创建态 WebUI 必须先扫码生成 `preview_id`，再用专用 browse API 选择目录 `cid`。不要让管理员手填 Cookie 或手猜 `cid`。

### 创建态扫码流程

```text
POST /api/manage/pan115/qr-login
  -> { session_id, uid, qr_url, qr_image? }

GET /api/manage/pan115/qr-status?session_id=...
  -> { status: waiting|scanned|signed|expired|canceled|aborted|unknown }

POST /api/manage/pan115/previews
  body: { session_id, cookie_app? }
  -> { preview_id }

POST /api/manage/pan115/previews/{previewId}/browse
  body: { path? }
  -> Pan115BrowseResponse

POST /api/manage/mounts
  -> mount detail

POST /api/manage/pan115/activate
  body: { mount_id, preview_id, cookie_app? }
  -> { ok: true, mount_id }
```

创建态预览是短 TTL 进程内状态。服务重启或预览过期后，皮肤应允许用户重新扫码，不应丢弃已经填写的 mount 表单。

### 媒体库绑定子路径仍是相对路径

虽然挂载根是 `cid`，但媒体库绑定 `sub_path` 仍然保存为相对 mount 根的逻辑路径：

```jsonc
{
  "mount_id": "mount_pan115_movies",
  "sub_path": "电影/4K",
  "scan_priority": 10
}
```

## `pan115-share` 端点

### 分享 Cookie 预览和绑定

| Method | Path | 说明 |
|--------|------|------|
| POST | `/api/manage/pan115/share-download-preview/qr-login` | 创建分享数据源前申请扫码 |
| GET | `/api/manage/pan115/share-download-preview/qr-status?session_id=` | 创建态扫码状态 |
| POST | `/api/manage/pan115/share-download-preview/create` | 将扫码会话转为短 TTL `preview_id` |
| POST | `/api/manage/pan115/share-mounts/{mountId}/qr-login` | 已有分享 mount 重新扫码 |
| GET | `/api/manage/pan115/share-mounts/{mountId}/qr-status?session_id=` | 已有分享 mount 扫码状态 |
| POST | `/api/manage/pan115/share-mounts/{mountId}/activate` | 绑定分享 Cookie；支持 `session_id` 或 `preview_id` |
| GET | `/api/manage/pan115/share-mounts/{mountId}/credentials` | 查看 mount 级分享 Cookie 状态 |
| DELETE | `/api/manage/pan115/share-mounts/{mountId}/credentials` | 解绑 mount 级分享 Cookie |

内部 API 路径仍保留 `share-download` 历史命名，但用户界面和文档语义统一叫 **115 分享 Cookie** 或 **挂载级分享 Cookie**。

### 多分享项 mount 配置

`pan115-share` 是媒体来源 provider。一个 mount 可以维护多个分享项；每个分享项的 `alias` 是媒体库可见的一级虚拟目录。

```jsonc
{
  "provider_type": "Pan115Share",
  "root_path": "/",
  "config_json": {
    "client_type": "chrome",
    "shares": [
      {
        "id": "share_item_a",
        "alias": "电影合集",
        "share_code": "abc123",
        "receive_code": "qwer",
        "sort_order": 0,
        "enabled": true
      },
      {
        "id": "share_item_b",
        "alias": "纪录片",
        "share_code": "https://115.com/s/xxxx",
        "receive_code": "",
        "sort_order": 1,
        "enabled": true
      }
    ],
    "request_policy": {
      "request_timeout_ms": 30000,
      "min_request_interval_ms": 500
    }
  }
}
```

关键语义：

1. `root_path` 固定为虚拟根 `/`，不是分享内子目录，也不是 share cid。
2. `shares[].alias` 必填且在同一个 mount 内唯一；它会成为一级目录名。
3. `share_code` 支持完整分享链接或纯 code，皮肤可自动提取。
4. `receive_code` 对公开分享可留空，对加密分享必须支持填写。
5. `client_type` 决定分享下载链路通道，默认 `chrome`。
6. 同一 `pan115-share` mount 下多个分享项共用一份 mount 级分享 Cookie；不同 mount 之间绝不能共享。

### 匿名导入与播放 Cookie 边界

`pan115-share` 的目录浏览、全量导入和增量同步不依赖 Cookie。只有以下动作需要 mount 级分享 Cookie：

- 生成播放 / 下载临时链接
- 技术探测
- sidecar 读取
- 资源缓存读取

缺少分享 Cookie 时，皮肤仍应允许创建和导入分享目录；播放或探测失败时应引导用户去当前数据源详情抽屉绑定分享 Cookie。

## `request_policy`

115 相关 provider 与其它远端源保持统一：前端只写 `config_json.request_policy`。

当前主线真实消费边界：

- `pan115`：runtime 注入时消费 `request_timeout_ms`、`min_request_interval_ms`
- `pan115-share`：分享目录浏览、播放和下载链路消费 `request_timeout_ms`、`min_request_interval_ms`

`max_retries` / `cooldown` 等其它治理字段目前仍是前向兼容字段，主题包可以展示，但不能声称当前版本后端已全部消费。

## 皮肤实现建议

1. `pan115` 创建态只提供扫码，不提供手填 Cookie。
2. `pan115-share` 创建态应先维护 `shares[]`，可选扫码生成分享 Cookie 预览；创建 mount 成功后自动绑定。
3. `pan115-share` 详情 / 编辑页必须提供分享 Cookie 状态、重新扫码和解绑。
4. 媒体库绑定页应在 `pan115-share` 的虚拟根下选择 `/别名/子目录`。
5. 详情页应显式区分：
   - `pan115`：账号状态、健康状态、Open token 刷新、解绑、raw sync 概览。
   - `pan115-share`：多分享项配置、挂载级分享 Cookie、raw sync 概览。
