# Manage · Pan115 (网盘账号 / 浏览)

115 网盘 provider 的管理面：扫码登录 → 激活 mount → 浏览目录 → 健康/刷新/解绑。

> 实施背景：rust-p115client SDK + `Pan115Provider` 静态链入 fmby。详细见 stage12U 同期产出的 `provider-115-native.md`。

## 端点

| Method | Path | 说明 |
|--------|------|------|
| POST   | `/api/manage/pan115/qr-login` | 申请二维码（`uid`、`qrcode_url`、`expire_at`） |
| GET    | `/api/manage/pan115/qr-status` | 长轮询扫码状态（`status: waiting|scanned|confirmed|expired|cancelled`） |
| POST   | `/api/manage/pan115/activate` | 把已确认的 uid 绑定到一个 mount，落库 |
| GET    | `/api/manage/pan115/accounts/{mount_id}` | 该 mount 的账号详情（脱敏） |
| POST   | `/api/manage/pan115/accounts/{mount_id}/refresh` | 刷新 access_token（OAuth Open） |
| POST   | `/api/manage/pan115/accounts/{mount_id}/health` | 主动健康探测：能否调 `/files` |
| DELETE | `/api/manage/pan115/accounts/{mount_id}` | 解绑（清密钥；不影响已索引文件） |
| POST   | `/api/manage/pan115/accounts/{mount_id}/browse` | 目录浏览（mount 感知） |

## 关键 DTO

`QrLoginReq`：`{ cookie_app?: "alipaymini|wechatmini|qandroid|harmony|115ios|115ipad|tv|web" }`（默认 `alipaymini`）  
`QrLoginResp`：`{ uid, qrcode_url, expire_at }`  
`QrStatusResp`：`{ status, message? }`，长轮询超时返回 `waiting`  
`ActivateReq`：`{ uid, mount_id }`  
`Pan115Account`：`{ mount_id, user_id_115, nickname, avatar_url, capacity_bytes, status: "active|expired|unbound", bound_at, last_health_at }`  
`BrowseReq`：`{ path: "/", page_token? }` → `BrowseResp { entries: [{ name, path, is_dir, pickcode?, size?, modified_at? }], next_page_token? }`

> 权威：`crates/fmby-api/src/manage/dto/pan115.rs`、`crates/fmby-provider-115/src/auth.rs`。

## 关键流程

```
qr-login -> uid+url    展示二维码
   |
   v
qr-status (长轮询)     scanned -> confirmed
   |
   v
activate(uid, mount)   落库 + 解密 KEK + 启动 runtime
   |
   v
browse / refresh / health / unbind
```

`cookie_app` 选项见 [domains/settings.md `features.pan115`]，默认 `alipaymini` 会挤占用户日常小程序登录态，皮肤应让用户自选（Stage 9 已实施）。

## 错误

- `400 bad_request`：uid 不合法
- `408 request_timeout`：长轮询超时（皮肤要"重试"而非"失败"）
- `409 conflict`：mount 已绑其它账号
- `410 gone`：uid 过期，需重新申请
- `412 precondition_failed`：refresh 失败需要重扫码
- `502 upstream_error`：115 上游问题
- `503 service_unavailable`：SDK 未初始化（KEK 缺失）

## 皮肤实现建议

- 二维码组件：用 `<select>` 让用户挑 cookie_app（默认 alipaymini）+ 倒计时 + 自动刷新
- 长轮询用 self-rescheduling setTimeout（不要 setInterval；HTTP 局域网 fetch 可能阻塞）
- 状态变 `confirmed` 后直接调 activate；不要在前端缓存 uid 跨页
- 详情卡：头像 + 昵称 + 容量进度条 + 4 按钮（刷新 / 检测 / 解绑 / 重扫）
- 浏览页：上一级 / 刷新 / 文件夹下钻；不展示文件大小给非 admin
- HTTP 局域网下复制 pickcode 走 navigator.clipboard fallback
